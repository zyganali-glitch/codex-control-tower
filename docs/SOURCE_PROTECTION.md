# Source Protection and Lineage Record

Codex Control Tower is an independent OpenAI Build Week project. The four Universal Agent OS repositories below were used only as protected, read-only research sources. They were not used as the working repository, package identity, or demo target.

Research was performed through public web pages and raw, read-only file views on 2026-07-12. No source repository was cloned into this workspace, modified, renamed, deleted, or pushed to.

## Protection lock

- Read public source pages only; never write to a source remote or a local checkout of it.
- Never use a protected source as a scan demo, generated-work target, package repository, or deployment target.
- Keep the active package name, metadata, commands, UI identity, and reports specific to `codex-control-tower`.
- Use only `examples/messy-saas-before`, `examples/governed-saas-after`, or another explicitly chosen target for demonstrations. InvoiceFlow Mini is controlled **SIMULATED DEMO DATA**, not a donor or customer project.
- Do not import donor screenshots, competition branding, hosted endpoints, credentials, tokens, private URLs, or private evidence.
- Do not infer permission from a permissive license. MIT compatibility permits reuse subject to its terms; it does not remove the need for attribution, source safety, or an originality review.
- If future work copies a substantial protected-source fragment, record the exact file, license, attribution, transformation, and destination here before release. The current record is: **no such copying**.

## Independent identity check

- Workspace: separate `codex-control-tower` directory.
- Product: Codex Control Tower — mission control for AI-built software.
- Active package: Codex Control Tower only; donor package metadata is not imported.
- Demo: InvoiceFlow Mini only, explicitly simulated.
- Runtime dependency on protected sources: none.
- Write operations performed against protected sources during this build: none.
- Active local Git remote: none; no donor or competition repository is configured as a remote.

## Claim discipline

The GitLab edition's reported competition placement was supplied as background. A public project-gallery listing was found, but no authoritative result page confirming a second-place award was located during this review. Codex Control Tower therefore makes **no award claim** about that source.

Repository self-descriptions were treated as source-authored claims, not independent proof. Where a source distinguishes implemented, live, mock, simulated, planned, or unavailable behavior, this research preserves that distinction.
