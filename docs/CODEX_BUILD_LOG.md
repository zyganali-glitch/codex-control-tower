## Important implementation decisions

- **Local-first and deterministic:** core scanning uses local structural/lexical signals; there are no hidden network calls or fake embeddings.
- **Content quality, not filename theater:** empty governance, test, CI, or evidence files cannot produce a high readiness score.
- **Scoped approval:** an `APPROVED` value without mission scope, allowed files, forbidden actions, note, and decision time does not unlock risky work.
- **Evidence vocabulary:** `PASS`, `WARN`, `FAIL`, `NOT_RUN`, and `SIMULATED` stay separate across JSON, Markdown, CLI, and dashboard.
- **Self-scan isolation:** generated evidence/Devpost packs and prior Control Tower JSON reports are excluded from later repository scoring.
- **Portable demo artifacts:** committed reports replace builder-specific absolute paths with repository-relative `./examples/...` paths.
- **Demo honesty:** the dashboard uses real local scans of two prepared fictional snapshots. Two fixture tests are executed by `cct demo`; GPT-5.6 does not create the snapshot difference. Fixture/provider browser/accessibility, load, deployment, and security checks remain `NOT_RUN`.
- **Product focus:** signed team gates, PR/MR integrations, and hosted collaboration remain future work; the real Blind GPT-5.6 Semantic Audit is implemented without giving the model authority to rewrite deterministic states.
- **Blind comparison, not hidden evidence:** raw evidence can contain status labels; the reconciler's locked claim-status fields and expected comparison classes are withheld from GPT-5.6.
- **Mechanically bounded model surface:** Codex CLI is pinned to `0.144.3` and launched in an empty ephemeral workspace. User/project instructions, web search, environment inheritance, and approval escalation are disabled; any command, file, MCP, web-search, plan, unknown, failed, or malformed event rejects the run.
- **Evidence-bearing model judgments:** `SUPPORTS` and `CONTRADICTS` require an allowed citation, `CONTRADICTS` requires counter-evidence, and `INSUFFICIENT` requires named missing evidence. Full-file hashes are kept distinct from hashes of compact/truncated bytes actually embedded for the model.
- **Structural versus semantic:** the local mission-alignment PASS means only that mission, trace, test execution, and evidence references structurally exist. GPT-5.6 supplies the semantic challenge; the local precheck is not represented as deterministic semantic truth.
- **Human escalation:** an assessment that conflicts with local comparison policy raises `HUMAN_REVIEW_REQUIRED`. `CONTRADICTS` alone is not always a conflict and never changes local truth.
- **Compatible uncertainty:** `INSUFFICIENT` against an already negative or unexecuted local state is `COMPATIBLE_WITH_LOCKED_STATUS`, not overstated as agreement and not treated as conflict.
- **Secondary exporter:** the Devpost Pack is an evidence/handoff convenience, not the core product value.

## Failures found and fixed

1. The initial workspace had no usable Git repository metadata. The independent local repository was later initialized on `main` with no remote, commit, or push.
2. PowerShell blocked `npm.ps1`; `npm.cmd` provided the same package operation without changing machine policy.
3. `evidenceSurfaceCount` was initially absent, producing invalid score arithmetic; explicit evidence-quality facts and regression coverage were added.
4. The first score model rewarded presence too strongly. An adversarial hollow repo could reach 97; content-quality checks reduced hollow governance below readiness. Release-preflight work then removed the opaque simulated-score cap and unearned Context Graph/Mistake Shield points while preserving a natural controlled `25 → 88` demonstration.
5. An unscoped gate could previously earn full approval points. Scope completeness and high-risk/out-of-scope blocking were added.
6. Evidence packs and prior reports could enter later scans and create recursive large-file risks. Generated markers and report-shape filtering made repeat scans stable.
7. The dashboard subtask could not build inside its narrower sandbox; the primary session ran the same Vite build successfully from the workspace root.
8. The first root suite correctly failed because this required log had not yet been created; the suite is rerun after documentation closure.
9. A later source-protection test correctly caught anthropomorphic wording even though it was negated. The wording was made structurally unambiguous, then the full suite passed.
10. Root dogfooding exposed three lexical/scope failures: README terminology implied simulation/staleness, controlled fixtures leaked risks, and package locks triggered monolith warnings. Explicit metadata, visible configured exclusions, comment-only markers, and source-only size checks fixed them.
11. Dashboard normalization supplied plausible-looking fallback values when evidence was absent. A source-derived adapter contract and five focused tests replaced those values with `UNKNOWN`, `null`, or no panel.
12. Mission Prompt publication rules conflicted with a scoped release approval. The prompt now carries the exact Review Gate scope and permits publication only when that scoped gate explicitly authorizes the action and destination.
13. The first live Codex task was too broad for a short demo. The contract was narrowed to bounded evidence and prohibited tests/search/progress updates. A historical low-reasoning run completed quickly; the final blind contract now uses medium reasoning for the higher-value semantic judgment.
14. The initial live dashboard report exposed the builder's absolute workspace path. Portable-path rewriting was added for the live report and Codex event stream before publication.
15. A no-command model contract initially supplied file names without file contents. A bounded, size-limited evidence bundle now provides the exact permitted local evidence as untrusted data, while the model remains read-only and cannot execute commands.
16. The compatibility table initially named Windows 11 without checking the host. The operating-system check showed Windows 10 Pro, and the documentation was corrected.
17. The first reconciliation UI treated only `SUPPORTS` as agreement, so a model correctly questioning a locked FAIL could look like disagreement. Semantic alignment now compares model assessment with the locked status meaning.
18. The earlier public narrative could imply that the model owned the overall verdict or created the demo improvement. The final local verdict is now deterministic, model opinion is separate, and prepared fictional snapshots are distinguished from real scans/tests/model execution.
19. The earlier recording path launched the model command from a second terminal and risked exposing the long build task. The featured path now uses a clean `Demo Blind Audit` Codex desktop task; the terminal is recovery only and `/feedback` comes privately from the separate primary build task.
20. The first hardened provenance run exposed a missing raw-test artifact because a repository-root path was interpreted inside the selected fixture target. GPT-5.6 surfaced the gap; demo generation now stores and references the raw output inside the target, and a regression assertion requires it.
21. External evaluation showed that the pre-blind prompt exposed the local status and expected answer class. The prompt was redesigned to withhold the reconciler's locked claim-status fields and expected comparison classes, while still exposing bounded raw evidence. Local reconciliation now owns the comparison policy and human-review flag.
22. A read-only sandbox rooted at the target repository still allowed model-initiated reads and instruction discovery. The final contract moved Codex into a newly created empty workspace, disabled user/project instructions and web search, removed inherited subprocess environment, pinned the CLI version, and added fail-closed JSONL validation that rejects every tool event.
23. The first mission-alignment local PASS could be misread as semantic proof, and `INSUFFICIENT` against a negative local state could be overstated as agreement. The claim is now explicitly a structural precheck, while reconciliation uses a third `COMPATIBLE_WITH_LOCKED_STATUS` relation for honest uncertainty.
