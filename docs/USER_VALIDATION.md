# User Validation Record

This document is a template for real, permission-based usability observations. It intentionally contains no fabricated users, quotes, adoption numbers, productivity claims, or outcomes.

## Test protocol

Ask each participant to use the public judge demo without prior coaching. Record only information they knowingly permit you to publish.

1. Can the participant explain the problem within 30 seconds?
2. Can the participant open and navigate the no-install demo?
3. Can the participant explain the Before / After difference?
4. Can the participant distinguish a locked deterministic state from the GPT-5.6 assessment?
5. Can the participant explain why NOT_RUN is not PASS?
6. Where did the participant hesitate or get stuck?
7. What single change would make the product easier to trust or use?

## Publication rule

Only summarize results in the README or Devpost submission after real sessions occurred. State the participant count, date range, tested commit, observed task, and limitations. Do not convert informal praise into a measured productivity or safety claim.
